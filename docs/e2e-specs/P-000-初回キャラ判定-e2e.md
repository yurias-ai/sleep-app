# P-000 初回キャラ判定（オンボーディング） E2Eテスト仕様書

対象: `index.html` `#view-onboarding`（オンボーディング制御スクリプト [5] / 共有ドメイン層 [3] / storage層 [1]）
前提共通: アプリURLを開く。storageは名前空間 `nemorino:` を使用。各テストは別途明記なき限り `nemorino:profile` 未保存の状態（初回）から開始する。

## 主要セレクタ / 実装事実
- 年/月/日: `#ob-year` `#ob-month` `#ob-day`（先頭は `value=""` のプレースホルダ「--」）
- 判定ボタン: `#ob-judge`（「判定する」）
- エラー表示: `#ob-error`（`hidden` 属性で表示制御）
- 結果カード: `#ob-result`（初期 `hidden`）、入力カード: `#ob-input`
- 結果要素: `#ob-emoji` `#ob-name` `#ob-personality` `#ob-rhythm`
- はじめるボタン: `#ob-start`
- トップ: `#view-top`（`.is-active` で表示）、下部ナビ: `body.onboarding` の間は非表示
- キャラ判定式（決定論）: `charaId = (((年+月+日) % 12) + 12) % 12`
- 保存形式: `nemorino:profile = {"birthdate":"YYYY-MM-DD","charaId":<0-11>}`
- 再訪スキップ判定 `isOnboarded()`: `profile && profile.charaId` が真のときスキップ（**charaId が 0 は falsy のためスキップされない**）

## 補足（実装由来の注意点・確認用）
- charaId 0（ねむり子鹿）になる生年月日（例: (年+月+日)%12==0）で判定→「はじめる」→リロードすると、`isOnboarded()` の `p.charaId` が 0=falsy のため**オンボーディングが再表示される**。これは現実装の挙動。E2Eで永続化を検証する際は charaId が 1〜11 になる生年月日（例 1990-04-15＝charaId 5）を用いること。
- 入力UIはセレクトのみで自由入力欄が無いため、XSS的な文字列入力テストは対象外。
- 「完全無料／料金は一切かかりません」の常時表示は P-000 に存在（`.badge-free` と `.note`）するが、本仕様の主対象はキャラ判定フローであり表示確認は補助的に E2E-P000-01 のスクリーンショットで担保する。
