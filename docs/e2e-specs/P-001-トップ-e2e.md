# P-001 トップ E2Eテスト仕様書

対象: 単一HTMLアプリ「眠りの森学園 体験版」`index.html` の P-001「トップ」（`#view-top`）
基づく実装: 共有ドメイン層 `Domain`、ルーター `App`、トップ描画スクリプト [6]（`render`/`renderChara`/`renderPhase`）

## 前提・データ準備の共通メモ
- storage 名前空間は `nemorino:`。`Storage` 経由で読む。
- profile キー: `App.PROFILE_KEY = "profile"`、形式 `{ birthdate, charaId }`。
- 生理開始日キー: `Domain.PERIOD_KEY = "periodStarts"`、形式 `["YYYY-MM-DD", ...]`（最新日を採用）。
- フェーズ判定: 開始日=1日目。経過日数 d で `d<=5 月経期` / `6..13 卵胞期` / `14..16 排卵期` / `17以降 黄体期`。
- 「今日のひとこと」（§13.1 刷新・**毎日変化**）= `Domain.getDailyMessage(charaId, phaseId, birthdate, today)`。
  - 生成式 = 今日のリズム導入（`getDailyRhythm(birthdate, today).lead`・誕生日バイオリズム風）＋ フェーズ別アドバイスの**変奏**（`pickPhaseAdvice(phaseId, today)` が `PHASE_MESSAGES[phaseId]` 配列から当日インデックス `dayIndex(today)` で1つ選択）＋ キャラトーン（`CHARA_TONES[charaId]`）。
  - 決定論: 同じ誕生日・フェーズ・キャラ・**同じ日**なら必ず同一文。**日が変わると**リズム導入とアドバイス変奏が変化し別日は別文になりうる。
  - 閉経時は `Domain.getMenopauseMessage(charaId, birthdate, today)`（リズム導入＋`MENOPAUSE_MESSAGES` 変奏＋キャラトーン・フェーズなしでも毎日変化）。
- 「今日」依存テストは固定の生年月日・生理開始日ではなく、テスト実行日からの相対日付（下記 D±）で開始日を投入して経過日数を確定させること。
  - D0 = 今日 → 経過1日（月経期）／ D-5 → 6日（卵胞期）／ D-13 → 14日（排卵期）／ D-16 → 17日（黄体期）／ D-30 → 31日（黄体期）。
- 日替わり検証で「別日」を作るには、`getDailyMessage`/`getDailyRhythm`/`dayIndex` に渡す `today` 引数（第4引数）へ別の日付を与える（純関数・副作用なし）。実 storage を汚さず決定論で比較できる。

## 注記
- P001-04〜08・11・12・15〜17 の経過日数・日替わり検証は実行日基準の相対日付・`today` 引数で扱い、固定日付を埋め込まないこと（「今日」依存のため）。
- 今日のひとことの完全一致／合成検証は `Domain.getDailyMessage` / `getMenopauseMessage`（および `PHASE_MESSAGES` / `CHARA_TONES` / `MENOPAUSE_MESSAGES` / `RHYTHM_STATES`）を真実の源として参照し、テスト側に文字列を二重定義しない。
- 日替わり（P001-15/17）は純関数へ `today` を渡して別日を再現し、storage を汚さずに検証する。
- もりのこ widget・クイズ誘導は §13.8 の実装（`renderGrowWidget` / `top-quiz-nudge`）に基づく。クイズ本体（出題・採点）は P-004 側で検証し、ここではトップの表示と導線のみを対象とする。
