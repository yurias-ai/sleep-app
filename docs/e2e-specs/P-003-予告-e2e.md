# P-003「予告」ページ E2Eテスト仕様書

生成日: 2026-06-15
対象ページ: 予告タブ（`#view-forecast` / `data-view="forecast"`）
種別: 単一HTML静的アプリ（ログイン・認証なし・外部送信なし）

---

## ユーザーゴール

1. 体験版で「おあずけ」になっている上位機能（4つ）の存在を予告として知れる
2. LINE登録で受け取れる特典内容と登録先IDを確認できる
3. 自分の状態（ゆらぎやすい時期／記録の蓄積）に応じてLINE案内が自然に強調され、登録動線へ進める

---

## テスト環境

```yaml
URL: http://localhost:3384/index.html
認証: なし（オンボーディング完了済みが前提。未完了なら生年月日判定→「はじめる」で完了させる）
データ: 端末内のみ（localStorage / メモリ state）。外部API・外部送信なし
ナビ: 下部3タブ（🏠トップ / 📝記録 / 🔒予告）。予告タブで対象ページへ遷移
キー: nemorino:profile / nemorino:periodStarts / nemorino:sweetCravings
強調条件: 黄体期（最新生理開始日から17日目以降） または 生理開始日＋甘いもの欲の記録合計が3件以上
```

> 前提セットアップ手順（各テスト共通）
> - オンボーディング: 生年月日を選び「判定する」→「はじめる」で完了（profile保存）
> - 状態を作る記録は「📝記録」タブで行う（生理開始日の入力・記録／甘いもの欲のタップ）
> - 予告ページは表示のたび `App.onShow("forecast", render)` で再評価される

## 補足: 外部送信なしの担保（全項目共通の非機能観点）

- いずれの操作でも外部へのネットワーク送信が発生しない（CSP `connect-src 'none'` により fetch/XHR/beacon は機構的に遮断）。
- LINEリンクは外部URLへの遷移リンクであり、ページからのデータ送信ではない（タップ時のみブラウザが遷移する）。
- データの読み書きは `nemorino:` 名前空間の localStorage（不可環境ではメモリ）に限定される。
