# P-004「そだてる」タブ E2Eテスト仕様書

対象: 単一HTMLアプリ「眠りの森学園 体験版」`index.html`
範囲: そだてる(P-004)タブ＝もりのこ育成(C)／睡眠クイズ(D)／ボーナスクイズ(E)／着せ替え(F)
前提原則: 完全無料・外部送信なし・決定論（同条件→同結果）・端末内storageのみ

> 実装事実ベース（推測なし）。日付は端末ローカル日付 `YYYY-MM-DD`。状態リセットは
> localStorage プレフィックス `nemorino:` のキー `grow / quiz / inventory / outfit / profile`
> を削除（アプリの `RESET_KEYS` と同一）。クイズの「翌日」検証は端末日付を進めるか、
> 該当キーの `lastYmd` / `lastOpenYmd` を直接書き換えて再描画する。

## B. もりのこ育成（C）

成長段階境界（`growthStage(openDays)`）: **1-2 たまご / 3-6 芽生え / 7-11 若葉 / 12-17 つぼみ / 18+ 満開**
`GROW_KEY="grow"` 形状: `{ startYmd, lastOpenYmd, openDays, collection:[{maturedYmd,stage}] }`

| ID | テスト項目 | 前提 | 操作手順 | 期待結果 |
|----|-----------|------|---------|---------|
| GR-GROW-01 | 初回はたまご | grow未生成 | そだてるを開く | `openDays` が1になり `#grow-stage-label`=「たまご」。`#grow-stage-step` に来訪日数を表示 |
| GR-GROW-02 | 毎日ひらくと育つ | grow初期 | 日付を1日ずつ進めながらそだて(またはトップ)を開く | openDays が +1 され、3で「芽生え」、7で「若葉」、12で「つぼみ」、18で「満開」に段階ラベルが切替 |
| GR-GROW-03 | 同日2回目は不変（決定論） | 当日すでに来訪済み | 同じ日にそだてる/トップを複数回開く | `lastOpenYmd===当日` のため openDays は増えない。段階ラベル・SVGが変わらない |
| GR-GROW-04 | 境界値: 2→3 で芽生え | openDays=2 (たまご) | 翌日に開く | openDays=3、ラベルが「たまご」→「芽生え」 |
| GR-GROW-05 | 境界値: 17→18 で満開 | openDays=17 (つぼみ) | 翌日に開く | openDays=18、ラベル「満開」。hero メッセージに「また明日ひらくと、新しい子がお庭に」系の案内 |
| GR-GROW-06 | 満開後の新たまご＋お庭増 | openDays=18 (満開) | 翌日に開く | collection に `{stage:"bloom"}` が1件追加、状態が新たまご(openDays=1,たまご)へ再スタート。`#grow-garden` に子が1体描画され `#grow-garden-empty` が隠れる |
| GR-GROW-07 | トップとそだてるで同期 | 任意の段階 | トップの今日のもりのこ表示とそだてるheroを比較 | `#top-grow-stage` と `#grow-stage-label` が同じ段階を示す（同一state参照） |

## C. 睡眠クイズ（D）

`QUIZ_BANK` 40問（q01-q40, 各 `difficulty 1易/2中/3難・question・choices[3]・answerIndex・explanation・onepoint`）
`QUIZ_KEY="quiz"` 形状: `{ lastYmd, answered:{}, correct:{}, bonusDone:{} }`。出題順=難易度昇順(易→難)＋決定論ハッシュ。

| ID | テスト項目 | 前提 | 操作手順 | 期待結果 |
|----|-----------|------|---------|---------|
| GR-QZ-01 | 1日1問の出題 | クイズ未回答 | そだてるを開く | `#grow-quiz-slot` に3択(`.qz-choice`)が1問だけ表示。進捗 `.qz-progress` が「N / 40」 |
| GR-QZ-02 | 易→難の順序 | クイズ未回答(初日) | 初回出題内容を確認 | 最初の出題が難易度「易(1)」帯の問題（決定論で固定） |
| GR-QZ-03 | 正解で解説＋ワンポイント＋アイテム1個 | 当日未回答 | 正解の選択肢をタップ | `.qz-verdict.ok`、`.qz-result` に「解説」と「先生のワンポイント」、着せ替えアイテム1個を獲得（inventory +1） |
| GR-QZ-04 | 不正解でも解説表示・アイテムなし | 当日未回答 | 誤答の選択肢をタップ | `.qz-verdict.ng`、解説・ワンポイントは表示。アイテム獲得なし（inventory不変）。当該問題は後日復習で再挑戦可 |
| GR-QZ-05 | 当日2問目は出ない | 当日に1問回答済み(正/誤問わず) | そだてるを再表示 | `lastYmd===当日` のため新規問題は出ず「また明日」系カードを表示 |
| GR-QZ-06 | 翌日は別問題 | 前日に回答済み | 日付を翌日へ進めてそだてるを開く | 前日と異なる未回答問題が1問出題される |
| GR-QZ-07 | 一巡後は復習モード | 全40問 answered・一部不正解 | そだてるを開く | 未正解問題が「復習・」バッジ付き（`.qz-cat.is-review`）で再出題される |
| GR-QZ-08 | 全問正解で gold_crown＋お祝い | 39問正解済み・最後の1問が当日出題 | 最後の問題に正解 | 40問正解達成。`gold_crown`(金の花冠)を獲得、`.qz-complete` に「全40問コンプリート」お祝い表示 |

## D. ボーナスクイズ（E）

`BONUS_BANK` 5問。節目→問題: 孵化 sprout(b01,openDays≥3) / 若葉 leaf(b02,≥7) / つぼみ bud(b03,≥12) / 満開 bloom(b04,≥18) / 新しい子 rebirth(b05,collection≥1)。
出現条件: 節目到達 かつ `bonusDone[milestone]` 未済。正解でアイテム2個。

| ID | テスト項目 | 前提 | 操作手順 | 期待結果 |
|----|-----------|------|---------|---------|
| GR-BN-01 | 節目で出現 | openDays=3到達(孵化)・該当bonus未済 | そだてるを開く | `#grow-quiz-slot .qz-bonus`（金枠・「✨ボーナスクイズ✨」）が通常クイズと共に出現 |
| GR-BN-02 | 正解でアイテム2個 | ボーナス出現中 | 正解の選択肢をタップ | `.qz-verdict.bonus`「正解！ボーナスで2つ受け取りました」、`.qz-reward-multi .qz-reward` が2件。inventory +2 |
| GR-BN-03 | 同節目で再出現しない | 当該節目のボーナス回答済み(正/誤問わず) | そだてるを再表示 | `bonusDone[該当milestone]=1` のため同じ節目のボーナスは再出現しない |
| GR-BN-04 | 不正解でも消化 | ボーナス出現中 | 誤答をタップ | 不正解表示。アイテム獲得なし。`bonusDone` は記録され再出現しない |
| GR-BN-05 | 各節目で順次出現 | 育成を進めて若葉→つぼみ等へ | 各節目到達時にそだてるを開く | leaf/bud/bloom/rebirth それぞれの節目で対応ボーナスが1回ずつ出現 |

## E. 着せ替え・アイテム（F）

`ITEM_CATALOG` 14点（通年9・季節4 spring/summer/autumn/winter・限定1 `gold_crown` special）。
`currentSeason`: 3-5春 / 6-8夏 / 9-11秋 / 12・1・2冬。1スロット(head/ear/neck/hand)につき着用1点。

| ID | テスト項目 | 前提 | 操作手順 | 期待結果 |
|----|-----------|------|---------|---------|
| GR-DU-01 | アイテム装着→もりのこ反映(hero) | 通年アイテムを所持 | `#du-grid` の所持タイルをタップ | タイルが `.is-worn`、`#grow-hero-art` のSVGに装飾が重なる |
| GR-DU-02 | トップにも反映 | アイテム装着済み | トップを開く | `#top-grow-art` にも同じ装飾が反映（hero/トップ共通の着用状態） |
| GR-DU-03 | 解除 | アイテム着用中 | 着用中タイルを再タップ | `.is-worn` が外れ、hero/トップから装飾が消える |
| GR-DU-04 | 同スロットは1点のみ | 同スロット(例head)を2点所持 | 1点目を着用後、2点目をタップ | 2点目が着用され1点目は自動的に外れる（slot最大1） |
| GR-DU-05 | 未獲得アイテムはロック | 未獲得アイテムあり | `#du-grid` を確認 | 未獲得タイルは `.is-locked`＋`disabled`、🔒(`.du-lock-key`)表示でタップ不可 |
| GR-DU-06 | 限定 gold_crown のロック | 全問正解未達 | グリッドの gold_crown タイルを確認 | `special` のため「全問正解で解放」ラベルでロック。全40問正解後に解放・装着可 |
| GR-DU-07 | 季節アイテムは季節一致時のみ獲得対象 | 端末が夏(6-8月) | クイズ正解でアイテムを獲得 | 自動付与対象は通年＋夏(summer)アイテムのみ。春/秋/冬アイテムは付与されない（タイル自体は通年表示・ロック） |
| GR-DU-08 | 季節見出しの表示 | 任意 | そだてるの着せ替え見出しを確認 | `#du-now-season` に端末日付由来の現在の季節が表示 |
