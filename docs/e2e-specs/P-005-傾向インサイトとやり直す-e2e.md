# P-005 傾向インサイト＆やり直す E2Eテスト仕様書

対象: 単一HTMLアプリ「眠りの森学園 体験版」`index.html`
対象機能:
- §13.7 あなたの傾向インサイト（記録タブ `#view-record` グラフ下のカード `.rec-insight`）
- やり直す（トップ `#view-top` 最下部 `#top-reset-link` → 2段階確認 → オンボーディング復帰）

## 前提・データ準備の共通メモ
- storage 名前空間は `nemorino:`。`Storage` 経由で読み書きする。
- 関連キー: `profile` / `Domain.PERIOD_KEY="periodStarts"` / `Domain.SWEET_KEY="sweetCravings"` / `Domain.MENOPAUSE_KEY="menopause"`。
- 甘いもの欲レコード形式: `{ date:"YYYY-MM-DD", level:1〜4 }`（`Domain.SWEET_LEVELS.length=4`）。`isSweetLevel` は 1〜4 の整数のみ有効。
- 傾向インサイト純関数: `Domain.computeSweetInsight(records, starts, menopause, today)`（決定論・副作用なし）。
  - 有効レコード（`isSweetLevel(level)` を満たす）が `INSIGHT_MIN=5` 件**未満**なら `{ enough:false, count, need:5 }`。
  - 5件以上かつ閉経でなく最新生理開始日があるとき: フェーズ別平均レベルを集計し、最大平均のフェーズ `topId` を採用。`{ enough:true, menopause:false, topPhase, message }`。
  - 閉経時、または最新生理開始日が無くフェーズ分類できないとき: フェーズ非依存（全体平均→`sweetTendencyWord`）の `{ enough:true, menopause:true, message }`。
- 表示側 `renderInsight()`:
  - `enough:false` → `#rec-insight-body` に「あと N 回ほど甘いもの欲を記録すると、傾向が見えてきます🌿」（N=need-count）。`#rec-insight-nudge` は hidden（導線なし）。
  - `enough:true` → `#rec-insight-body` に `r.message`。`#rec-insight-nudge` 表示（`#rec-insight-nudge-text` に導線文＋`#rec-insight-link`「→ LINE / 体験入学を見る」`href=https://s.lmes.jp/landing-qr/2006353028-ZkXRAR0K?uLand=HfZHgJ`）。
- フェーズ判定（開始日=1日目）: `d<=5 月経期` / `6..13 卵胞期` / `14..16 排卵期` / `17以降 黄体期`。
- フェーズ別平均の集計は各レコードの `date` を最新開始日基準で分類する。「今日」依存を避けるため、生理開始日・甘いもの欲レコードはテスト実行日からの相対日付（D±n）で投入し、狙ったフェーズへ確定させること。
- やり直す `RESET_KEYS` = `profile` / `periodStarts` / `sweetCravings` / `menopause` / `grow` / `quiz` / `inventory` / `outfit`（`nemorino:` 名前空間の全アプリキー）。
- `#top-reset-link` クリック → `#top-reset-confirm` を表示（hidden解除）しリンクを隠す。確認カード内 `#top-reset-no`（やめておく）／`#top-reset-yes`（はい、やり直す）。
- `#top-reset-yes` → 全 `RESET_KEYS` 削除 → `window.location.reload()`（失敗時フォールバックで `body.onboarding` 付与＋オンボーディング表示）。
- トップ表示のたび（`App.onShow("top")`）確認カードは閉じる（`showConfirm(false)`）。

## 注記
- 傾向の検証は `Domain.computeSweetInsight` を真実の源とし、テスト側に文言を二重定義しない（フェーズ label は `Domain.PHASES[id].label` を参照）。
- フェーズ依存ケース（INS-04/05/07）は実行日基準の相対日付でレコード・開始日を投入し、固定日付を埋め込まないこと。
- `RESET_KEYS` の真実の源は reset スクリプトの定数（`App.PROFILE_KEY` ＋ Domain の各 KEY）。テストはこの8キーが消えることを確認する。
- 実装に無い文言・分岐（フェーズ非依存時に○○期を出す等）はテストしない。
