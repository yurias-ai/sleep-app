## 1. プロジェクト概要

### 目的・位置づけ
女性起業家専門睡眠コーチ「眠りの森学園」が、LINE登録者・個別相談申込者に**無料プレゼント**する体験版アプリ。本番の有料講座（睡眠スコア・食事アップロード・行動記録）への"入口"として機能させる。アプリ単体で全てを解決させず、「もっと整えたい」と思わせて講座へ橋渡しするのが狙い。

### 配布導線
```
【LINE登録者】 リッチメニュー →「睡眠診断アプリ」タップ → アプリURLを開く（LINE内ブラウザ想定）
【Instagram等】 投稿/プロフ →「こちら」タップ → LP（ブラッシュアップ予定）→ 個別相談申込 → 特典としてアプリURLを渡す
```
→ **安定した固定URL**で開ける必要がある（リッチメニュー＆特典配布の両方でURLを使う）。

### 成功指標
**定量指標**
- 毎日の操作はタップ1〜2回で完了（必須入力は3項目以内）
- 主要画面は3画面 + 初回キャラ判定
- ランニングコスト 月0円（サーバー費・API費・課金・サブスク すべてゼロ）

**定性指標**
- 「もっと整えたい」と思わせ、LINE/講座へ自然に橋渡しする
- 助産師×睡眠コーチの専門的かつ温かいトーン
- ベージュ×淡グリーンの癒し世界観で離脱させない軽さ

## 2. 最重要の制約：完全無料・料金ゼロ（絶対遵守）

- サーバー費用・API利用料・課金・サブスクが**一切発生しない**構成にすること
- 有料の外部API（生成AI API、有料DB等）を**呼び出さない**
- ユーザーにも開発者にも料金請求が発生しない
- データ保存は**端末内のみ**で完結（storage利用可なら localStorage/sessionStorage、不可ならアプリ起動中のメモリ内 state で保持）。外部サーバー・クラウド保存は使わない
- ログイン不要・アカウント登録不要・個人情報をサーバー送信しない
- 画面のどこかに「**完全無料／料金は一切かかりません**」と明記

> 実装方針：**単一HTMLファイル（自己完結）** を無料静的ホスティング（GitHub Pages 等）に置いて固定URL化。外部リクエストはゼロ。

## 4. 画面詳細仕様

### P-000：初回キャラ判定（初回1回のみ）
- 生年月日を入力（年・月・日のセレクト/タップ）
- 「判定する」タップ → **動物キャラ**を表示（独自オリジナルキャラ）
- キャラごとの性格・リズム傾向を一言で表示
- 判定は**外部APIを使わず内部の決定論ロジック**で行う（§7 参照）
- 判定後はキャラ情報を端末内に保持し、次回以降はこの画面をスキップしてトップへ
- ⚠️ **「個性心理学」という語・出典はアプリ内に一切出さない**（独自の動物キャラとして表現）

### P-001：トップ
- 今日の**月経フェーズ**（月経期／卵胞期／排卵期／黄体期）を表示
- **動物キャラ**を表示
- **今日のひとこと**：現在フェーズ × 動物キャラを掛け合わせた、睡眠・体調・甘いもの欲に関する短いアドバイスを1つ表示（助産師×睡眠コーチの温かい専門トーン）
- 「**完全無料／料金は一切かかりません**」を明記
- 注意書き（§9）を表示
- 記録が未登録の場合は「記録画面へ」の優しい誘導

### P-002：記録
- **生理開始日の記録**：「生理が始まった日」を日付タップで記録するだけ（詳細入力不要）。記録から現在フェーズを自動判定
- **甘いもの欲レベルの記録**（任意）：1日1回、3段階をタップ記録
- **甘いもの欲グラフ**：黄体期に欲が上がる傾向を可視化する簡単なグラフ（外部ライブラリ非依存の軽量描画 or 軽量CSS/SVG）
- すべての詳細記録は**任意（オプション）**扱い

### P-003：予告（"おあずけ"機能）
以下は**実装せず**、「講座に入るとここまで分かります」という予告カード（鍵マーク・ロック表示）として表示：
- 睡眠スコア自動算出
- 食事の写真アップロード分析
- 朝のスッキリ感チェック
- 行動記録と睡眠の相関分析

**LINE導線**：
- 黄体期に入ったタイミング、または数日記録が貯まったタイミングでLINE案内を表示（予告画面＋トップにも自動表示）
- 特典文言（そのまま使用）：
  ```
  📲LINEに登録すると今すぐ受け取れます🌿
  🧭 睡眠の羅針盤（独自の睡眠診断）
  🌱 今夜から使えるケア6選
  🎓 眠りの森学園 体験入学
  → @l.hidamari
  ```

## 5. データ設計概要（端末内のみ／サーバー送信なし）

| エンティティ | 項目 | 保持先 |
|-------------|------|--------|
| ユーザープロフィール | 生年月日、判定済み動物キャラID | storage可:localStorage / 不可:メモリ state |
| 月経記録 | 生理開始日（複数可、最新を採用） | 同上 |
| 甘いもの欲記録 | 日付, レベル(1〜3) の配列 | 同上 |

- **storage抽象レイヤー**を1つ用意：`localStorage`が使える環境なら永続化、`try/catch`で失敗したらメモリ内Mapにフォールバック（LINE内ブラウザ等での制限を吸収）。
- バリデーション：生年月日は実在日付チェック、未来日は不可。生理開始日は未来日不可。

## 6. 月経フェーズ判定ロジック（決定論）

標準28日周期を仮定し、最新の生理開始日からの経過日数 `d`（開始日=1日目）で判定：

| フェーズ | 経過日数 |
|----------|----------|
| 月経期 | 1〜5日目 |
| 卵胞期 | 6〜13日目 |
| 排卵期 | 14〜16日目 |
| 黄体期 | 17日目以降（次周期開始まで） |

- 経過日数が28を超えた場合も、新たな開始日の記録があるまでは「黄体期（周期が長引いている可能性）」として温かく案内。
- 生理開始日が未記録の場合は「まず記録してみましょう」の促し（フェーズ非表示）。

---

## 7. 動物キャラ判定ロジック（独自・決定論・著作権リスクなし）

- **完全オリジナル**の動物キャラ（例：12種）。「個性心理学」由来であることは一切表に出さない。
- 生年月日から**決定論的に**1キャラへ振り分ける独自計算式を用いる（例：`(年+月+日)` ベースの整数演算を12で割った剰余でキャラ確定 等。同じ生年月日なら必ず同じキャラ）。
- 各キャラに「名前・性格の一言・睡眠/リズムの傾向の一言」を持たせる。
- 具体的なキャラ一覧・文言、および「フェーズ×キャラ」の今日のひとこと文面は **Phase 4（ページ実装）前に本書のコンテンツ別紙 or 実装時に確定**（@01のページ深掘りで詰める）。
- 今日のひとことは「フェーズ別ベースメッセージ × キャラのトーン補正」の組み合わせで生成（全パターンを静的データとして内包、外部API不使用）。

## 8. 技術スタック

```yaml
形態: 単一HTMLファイル（自己完結 / 外部リクエストゼロ）
言語: HTML + CSS + Vanilla JavaScript（フレームワーク不要・ビルド不要）
グラフ: 軽量SVG/Canvas自前描画（外部ライブラリ非依存）
データ保存: localStorage（不可時はメモリ state にフォールバック）
ホスティング: 無料静的ホスティング（GitHub Pages 等）で固定URL化
ローカルプレビュー: 静的サーバ（port 3384）
コスト: 月0円（サーバー・API・課金すべてなし）
```

> 3画面＋初回判定という規模・「軽さ最優先」・「storage制限環境対応」・「完全無料」をすべて満たすため、React等のビルド系は採用せず**単一HTML**とする（最小性）。

## 11. セキュリティ要件

- 外部送信なし・サーバーなしのため攻撃面は最小。
- 入力値サニタイゼーション（生年月日・日付入力の妥当性チェック）。
- XSS対策：ユーザー入力をDOMへ挿入する際は `textContent` を用い、`innerHTML` への生挿入を避ける。
- 外部スクリプト・外部CDN・トラッキングを読み込まない（完全自己完結＝プライバシー保護にも寄与）。
- 認証・決済・個人情報のサーバー保持は存在しない（設計上の不要化）。
